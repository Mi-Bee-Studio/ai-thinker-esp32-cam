# 用户指南

本指南说明如何使用 AI_Thinker ESP32-CAM 固件，包括配置、Web 界面使用和功能设置。

## Web 界面访问

### 连接方式

#### STA 模式（推荐）
1. 设备自动连接到您的 WiFi 网络
2. 从路由器的设备列表中查找设备 IP，或检查串口监视器
3. 访问：`http://<设备-ip>/`

#### AP 模式（首次配置）
1. 连接到 WiFi 网络：**ai-thinker-cam**（密码：`12345678`）
2. 在浏览器中打开：**http://192.168.4.1/**

### Web 界面页面

#### 1. 仪表板（`/`）
主仪表板显示实时设备状态：

- **设备信息**：名称、运行时间、WiFi 状态
- **摄像头状态**：分辨率、FPS、JPEG 质量
- **存储信息**：SD 卡状态、可用空间
- **运动检测**：当前状态、事件数量
- **系统健康**：CPU 温度、内存使用
- **网络信息**：IP 地址、信号强度

#### 2. 预览（`/preview.html`）
带控制的实时 MJPEG 视频预览：

- **实时流**：实时摄像头画面
- **捕获按钮**：拍摄单张照片
- **分辨率选择器**：更改摄像头分辨率（VGA/SVGA/XGA/UXGA）
- **质量滑块**：调整 JPEG 质量（1-63，值越低质量越好）
- **下载按钮**：将当前帧保存到您的计算机

#### 3. 配置（`/config.html`）
完整的设备配置界面：

- **WiFi 设置**：SSID、密码、设备名称
- **摄像头设置**：分辨率、帧率、JPEG 质量
- **运动检测**：阈值、冷却时间
- **NAS 上传**：协议、服务器设置
- **时间设置**：时区、NTP 服务器
- **安全设置**：Web 密码（可选）

#### 4. 文件（`/files.html`）
照片文件管理：

- **照片库**：按日期组织的保存照片列表
- **下载**：单独下载照片
- **删除**：删除照片
- **上传到 NAS**：将照片发送到 NAS 服务器

## 配置选项

|---------|-------------|---------|-------|

| **JPEG 质量** | 图像质量（值越低质量越好） | 12 | 1-63 |

### 运动检测

| **保存照片** | 将运动照片保存到 SD 卡 | 启用 | 是/否 |

## LED 状态指南

状态 LED（GPIO33）指示不同的系统状态：

| 模式 | 图案 | 描述 |
|-------|---------|-------------|
| **常亮** | 启动中 | 系统正在启动 |
| **快速闪烁（0.5秒亮，0.5秒灭）** | WiFi 连接中 | 正在连接到 WiFi |
| **慢速闪烁（2秒亮，2秒灭）** | WiFi 已连接 | WiFi 已连接，正常运行 |
| **中等闪烁（1秒亮，1秒灭）** | AP 模式 | 接入点模式激活 |
| **红色快速闪烁（0.2秒亮，0.8秒灭）** | 错误 | 系统错误条件 |

## SD 卡使用

### 支持的格式
- **文件系统**：FAT16/FAT32
- **容量**：最多 32GB（推荐 8GB-16GB）
- **速度**：推荐 Class 10 或更高

### 文件组织
照片按以下目录结构存储：
```
/sdcard/photos/
├── 2024-12-30/
│   ├── 14-30-25.jpg
│   ├── 14-32-10.jpg
│   └── 14-35-42.jpg
├── 2024-12-31/
│   ├── 09-15-33.jpg
│   └── 09-18-05.jpg
└── motion_events/
    ├── 2024-12-30/
    └── 2024-12-31/
```

### 存储管理
- **自动清理**：当可用空间 < 20% 时，删除最旧的照片
- **手动清理**：使用 Web 界面删除特定照片
- **监控**：在仪表板上显示存储状态

## 运动检测

### 工作原理
运动检测使用帧差算法：
1. 捕获连续的 JPEG 帧
2. 转换为灰度并调整大小以便比较
3. 计算帧之间的像素差异
4. 如果差异超过阈值，触发运动事件

### 配置
- **阈值**：值越高灵敏度越低（1-255）
- **冷却时间**：运动事件之间的最小时间（秒）
- **区域**：比较帧的中心区域（640x480 → 320x240）

### 运动事件
当检测到运动时：
1. LED 表示运动（短暂闪烁）
2. 照片保存到 SD 卡
3. 触发 NAS 上传（如果启用）
4. 事件记录到串口输出

## NAS 上传

### 支持的协议

#### HTTP POST
- **端点**：`POST /upload`
- **标头**：`Content-Type: image/jpeg`，`X-Device-ID: <设备名称>`
- **主体**：JPEG 二进制数据
- **最适用于**：像 iot-server-base 这样的 Web 服务器

#### FTP
- **命令**：STOR 用于上传
- **模式**：被动模式
- **认证**：用户名/密码
- **最适用于**：传统 NAS 服务器

#### WebDAV
- **方法**：PUT 带认证
- **标头**：Content-Type 和授权
- **最适用于**：启用了 WebDAV 的云存储

### 上传工作流程
1. 检测到运动或手动触发
2. 捕获照片并保存到 SD 卡
3. 添加到上传队列
4. 重试机制：3 次尝试，间隔 2 秒
5. 成功：从队列中删除文件
6. 失败：稍后重试

## 命令行界面

### 使用 `curl` 进行 API 访问

#### 获取状态
```bash
curl http://192.168.1.100/api/status
```

#### 更新 WiFi 配置
```bash
curl -X POST http://192.168.1.100/api/config \
  -H "Content-Type: application/json" \
  -d '{"wifi_ssid":"MyNetwork","wifi_pass":"MyPassword"}'
```

#### 捕获照片
```bash
curl -o captured.jpg http://192.168.1.100/capture
```

#### 重启设备
```bash
curl -X POST http://192.168.1.100/api/reboot
```

### 串口监视器命令
通过 USB 连接（COM8）：

```
# 帮助命令
help

# 获取当前配置
get config

# 重启设备
reboot

# 恢复出厂设置
reset
```

## 常见问题排查

### WiFi 连接问题
- **检查凭据**：验证 SSID 和密码是否正确
- **仅限 2.4GHz**：确保您的路由器支持 2.4GHz WiFi
- **信号强度**：如果信号弱，移近路由器
- **AP 模式回退**：如果 STA 失败，设备自动进入 AP 模式

### 摄像头问题
- **无视频**：检查摄像头模块是否正确安装
- **图像失真**：检查摄像头镜头和连接
- **分辨率问题**：从 VGA 开始以获得稳定性
- **PSRAM 错误**：确保安装了 4MB PSRAM 且正常工作

### SD 卡问题
- **未检测到**：尝试不同的 SD 卡或重新插入
- **文件系统**：使用 FAT32 格式化
- **读写错误**：使用高质量的 SD 卡（Class 10+）
- **存储已满**：清理旧照片或使用更大的卡

### 运动检测
- **太灵敏**：增加阈值值
- **不够灵敏**：降低阈值值
- **误触发**：调整冷却时间
- **无检测**：检查摄像头对焦和照明

## 性能优化

### CPU 使用率
- **流传输**：VGA 分辨率下约 15 FPS，占用约 40% CPU
- **运动检测**：激活时约 5% CPU
- **空闲**：约 2% CPU 使用率

### 内存管理
- **空闲堆栈**：应保持 >20KB
- **PSRAM**：摄像头使用最多 2.4MB
- **SPIFFS**：Web 界面 ~500KB

### 电源节省
- **深度睡眠**：不支持（摄像头需要连续电源）
- **电源管理**：根据负载自动进行时钟调节
- **LED 控制**：为节省电源可以禁用状态 LED

## 安全考虑

### 网络安全
- **WiFi 加密**：推荐使用 WPA2
- **Web 界面**：密码保护是可选但推荐的
- **API 访问**：只读端点不需要认证

### 数据安全
- **照片**：存储在 SD 卡上本地
- **配置**：在 NVS 存储中加密
- **上传**：云上传推荐使用 HTTPS

### 物理安全
- **工厂重置**：BOOT 按钮用于紧急重置
- **LED 状态**：系统健康状态的视觉指示
- **温度监控**：自动热保护
